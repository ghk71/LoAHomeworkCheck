# Feature Spec

## 1. 공통

- 모든 페이지는 같은 디자인 톤을 유지한다.
- 화면 하단에 JavaScript 코드가 노출되면 안 된다.
- `</html>` 뒤에는 어떤 코드도 있으면 안 된다.
- CSS 변수 `var(--...)`를 쓰면 반드시 `:root`에 정의한다.
- GitHub Pages에서 별도 빌드 없이 동작해야 한다.

## 2. index.html

### 계정

- 계정 추가/수정/삭제
- 계정 숨김
- 계정 순서 드래그
- 부계정 설정
- 부계정은 본계정 아래 하위 탭으로 표시

### 캐릭터

- 캐릭터 추가/수정/삭제
- 캐릭터 순서 드래그
- 캐릭터 아이콘 선택
- 아이콘은 `icon_url`에 저장

### 원정대 숙제

- 추가/수정/삭제
- 아이콘 선택/제거
- 체크 상태 저장
- 다른 계정으로 복제

### 캐릭터 숙제

- 추가/수정/삭제
- 아이콘 선택/제거
- 하위 숙제 관리
- 일일/주간 초기화
- 체크 상태 저장

### 레이드 숙제

- 레이드명 + 난이도 표시
- 입장 레벨 표시
- 유통골드/귀속골드/더보기 골드 분리 표시
- 각 골드 항목에 hover 효과
- 각 골드 항목은 수령 여부를 별도로 토글
- 레이드 숙제 항목에서는 수정 버튼을 막음
- 파티연동 배지 표시
- 파티연동 팝업은 해당 캐릭터가 포함된 파티만 표시
- 파티 이동 버튼 제공

### 재화

- 재화 추가/수정/삭제
- 금액 직접 입력
- ± 버튼으로 금액 조정
- 아이콘 선택/제거
- 재화현황 중앙에 `최종 수정 날짜: YYYY.MM.DD` 표시

## 3. core.html

- 1700 이상 캐릭터 표시
- 기본 보기
- 번호별 묶음 보기
- 사용자 조합 보기
- 상단 보기 탭은 raid.html 탭 스타일과 유사하게 구성
- 마우스 휠 스크롤 가능해야 함
- 코어 등급 클릭 변경
- 필수 별표 토글
- 혼돈해1/혼돈달1/혼돈별1 텍스트 클릭 편집
- 변경된 혼돈 이름은 Supabase DB에 저장
- 사용자 조합 카드 크기는 과도하게 크지 않게 유지
- 등급 배지 hover 효과 제공

## 4. raid.html

- 기본 탭은 주간 일정
- 파티 구성 탭
- 레이드 프리셋 추가/수정/삭제
- 난이도 추가/수정/삭제
- 파티 추가/수정/삭제
- 파티 멤버 배치
- 입장레벨 필터
- 일정 추가/수정/삭제
- 주간 일정 완료/완료 해제
- 임시 파티 추가/삭제
- 원본 초기화
- 주차 이동
- 공지사항 표시
- 편집모드와 공유링크 뷰어모드 모두 공지가 잘 보여야 함
- 공유링크 뷰어모드에서 레이드 현황 탭 표시
- 공유링크 레이드 현황에는 캐릭터 필터 표시
- 공유링크에서는 완료 체크 불가, 상태 표시만 가능

## 5. overview.html

- 계정/부계정 그룹화
- 캐릭터별 레이드 완료 현황 표시
- 파티연동 팝업은 해당 캐릭터가 포함된 파티만 표시
- 완료 상태는 index.html 및 raid.html 완료 처리와 연동

## 6. parties.html

- 레이드명 옆에 난이도 표시
- 난이도 색상 유지
- 파티 정보 하단에서 난이도 중복 표시 제거
- 파티 멤버는 슬롯 순서대로 표시
- 빈 슬롯 표시
