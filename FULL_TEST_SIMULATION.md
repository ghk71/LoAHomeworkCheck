# Full Test Simulation

이 문서는 로스트아크 숙제 트래커의 전체 기능을 실제 사용자 흐름에 가깝게 검증하기 위한 시나리오 모음이다.

대상 파일:

- `index.html`
- `core.html`
- `raid.html`
- `overview.html`
- `parties.html`

목표:

- 과거 요청 기능과 최근 수정 기능을 모두 포함한다.
- 한 기능에서 파생되는 연동 기능, 새로고침 유지, 다른 페이지 반영까지 함께 확인한다.
- GitHub Pages 정적 HTML 구조와 Supabase 데이터 흐름을 깨뜨리는 회귀를 찾는다.

---

## 0. 테스트 준비

### 0-1. 환경 준비

1. GitHub Pages 또는 로컬 정적 서버에서 프로젝트를 연다.
2. 브라우저 개발자도구 Console을 연다.
3. Supabase에 `schema.sql` 최신 스키마가 적용되어 있는지 확인한다.
4. 테스트용 브라우저 프로필 또는 시크릿 창을 준비한다.
5. 기존 데이터가 많다면 테스트용 계정/캐릭터 이름 앞에 `TEST_`를 붙인다.

성공 기준:

- 모든 페이지가 빌드 과정 없이 직접 열린다.
- Console에 빨간 런타임 에러가 없어야 한다.
- Supabase 컬럼 오류가 없어야 한다.

### 0-2. 공통 정적 구조 검사

1. `index.html`, `core.html`, `raid.html`, `overview.html`, `parties.html`를 각각 연다.
2. 페이지 하단에 JavaScript 코드가 텍스트로 노출되지 않는지 확인한다.
3. 상단 메뉴 이동이 모든 페이지에서 정상인지 확인한다.
4. 디자인 톤이 크게 깨지지 않는지 확인한다.
5. 수정 후에는 `node tools/check-project.js`를 실행한다.

성공 기준:

- `</html>` 뒤에 코드가 없어야 한다.
- 사용한 CSS 변수는 모두 정의되어 있어야 한다.
- 공통 네비게이션이 정상 작동한다.
- Console 에러가 없다.

## 1. index.html 계정/부계정 시나리오

### 1-1. 본계정 생성, 수정, 숨김, 삭제

1. `index.html`에서 계정을 추가한다.
2. 계정명을 수정한다.
3. 계정 숨김을 켜고 목록에서 표시 상태를 확인한다.
4. 숨김 계정을 다시 표시한다.
5. 테스트용 계정을 삭제한다.

성공 기준:

- 계정 추가/수정/숨김/복구/삭제가 Supabase에 저장된다.
- 좁은 화면에서도 계정명, 캐릭터 수, 수정/숨김/삭제 버튼이 겹치지 않는다.
- 삭제 후 관련 캐릭터 선택 상태가 깨지지 않는다.

### 1-2. 부계정 생성과 본계정 하위 표시

1. 본계정 `TEST_MAIN`을 만든다.
2. 부계정 `TEST_SUB_A`, `TEST_SUB_B`를 만들고 `TEST_MAIN`의 부계정으로 설정한다.
3. 좌측 계정 탭에서 부계정이 본계정 아래 하위 탭으로 보이는지 확인한다.
4. 부계정 표시용 `...` 같은 임시 텍스트가 보이지 않는지 확인한다.
5. 계정 순서를 드래그해서 바꾼 뒤 새로고침한다.

성공 기준:

- 부계정은 독립 본계정처럼 중복 표시되지 않는다.
- 계정 순서가 새로고침 후 유지된다.
- 좁은 해상도에서도 버튼과 텍스트가 겹치지 않는다.

파생 확인:

- `overview.html`, `raid.html` 공유 뷰어, `parties.html`에서도 본계정/부계정 그룹이 일관되어야 한다.

## 4. index.html 캐릭터 숙제 시나리오

### 4-1. 일일 숙제 추가와 진행률

1. `TEST_CHAR_1`에 캐릭터 숙제를 추가한다.
2. 모달의 `그룹 선택`에서 `일일 숙제`를 선택한다.
3. 저장 후 하단 왼쪽 `일일 숙제` 영역에 표시되는지 확인한다.
4. 완료 체크한다.
5. 상단 캐릭터 탭 완료 상태와 하단 진행률이 즉시 갱신되는지 확인한다.
6. 새로고침 후 완료 상태가 유지되는지 확인한다.

성공 기준:

- 기본 추가값은 일일 숙제다.
- 상단 캐릭터 완료 상태는 일일 숙제 기준으로 계산된다.
- 완료 숨기기를 켜면 즉시 목록에서 사라진다.

### 4-2. 주간 숙제 추가와 분리 표시

1. 숙제 추가 모달을 연다.
2. `그룹 선택`에서 `주간 숙제`를 선택한다.
3. 초기화 요일을 수요일 외 다른 요일로 변경한다.
4. 저장 후 하단 오른쪽 `주간 숙제` 영역에 표시되는지 확인한다.
5. 주간 숙제를 완료 체크한다.
6. 상단 캐릭터 탭 완료 상태가 주간 숙제로 인해 완료 처리되지 않는지 확인한다.

성공 기준:

- 주간 숙제는 오른쪽 영역에 표시된다.
- 일일/주간 선택은 별도 그룹 선택 UI로 명확히 동작한다.
- 주간 숙제 완료는 하단 주간 영역 카운트에만 반영된다.

### 4-3. 하위 숙제와 상위 완료 상태

1. 일일 숙제에 하위 숙제 3개를 추가한다.
2. 하위 숙제를 하나씩 완료한다.
3. 모든 하위 숙제를 완료했을 때 상위 숙제 상태가 바뀌는지 확인한다.
4. 하위 숙제 하나를 완료 해제한다.
5. 상위 숙제 상태와 상단 캐릭터 탭이 즉시 갱신되는지 확인한다.

성공 기준:

- 하위 숙제 완료 상태가 상위 숙제와 동기화된다.
- 하위 숙제 변경 후 전체 목록과 상단 캐릭터 탭이 즉시 다시 렌더링된다.

### 4-4. 하위 숙제 드래그 정렬 회귀

1. 하위 숙제 5개를 만든다.
2. 완료 숨기기를 켠다.
3. 일부 하위 숙제만 보이는 상태에서 드래그 정렬한다.
4. 완료 숨기기를 끄고 전체 순서를 확인한다.
5. 새로고침 후 순서를 다시 확인한다.

성공 기준:

- 화면 인덱스가 아니라 task id 기준으로 순서가 이동한다.
- 완료 숨김/필터 상태에서 순서가 꼬이지 않는다.

### 4-5. 활성화 요일과 활성화 보이기

1. 주간 숙제에 활성화 요일을 설정한다.
2. 현재 표시되지 않는 요일 조건을 만든다.
3. `활성화 보이기`를 켠다.
4. 비활성 숙제가 표시되는지 확인한다.
5. `활성화 숨기기`를 눌러 다시 숨긴다.

성공 기준:

- 활성화 토글이 즉시 반영된다.
- 원정대 숙제의 활성화 토글과 캐릭터 숙제의 활성화 토글이 분리되어 동작한다.

## 5. index.html 레이드 숙제 시나리오

### 5-1. 레이드 숙제 표시와 골드 분리

1. 레이드 숙제를 추가한다.
2. 레이드명, 난이도, 입장 레벨, 유통골드, 귀속골드, 더보기 골드를 입력한다.
3. 목록에서 레이드명 옆 난이도 색상과 크기를 확인한다.
4. 유통/귀속/더보기 골드가 각각 분리된 뱃지로 보이는지 확인한다.
5. 각 골드 뱃지에 hover 효과가 있는지 확인한다.

성공 기준:

- 난이도 텍스트 색상과 크기가 `raid.html` 규칙과 어긋나지 않는다.
- 세 골드 항목이 하나로 합쳐지지 않는다.

### 5-2. 골드 수령 토글

1. 유통골드 수령 여부를 끈다.
2. 귀속골드 수령 여부를 끈다.
3. 더보기 골드 수령 여부를 끈다.
4. 새로고침한다.
5. `overview.html`로 이동해 골드 합계에 반영됐는지 확인한다.

성공 기준:

- `receive_gold`, `receive_bound`, `receive_bonus`가 각각 독립적으로 동작한다.
- 새로고침 후 상태가 유지된다.
- overview 골드 집계와 index 잔여 골드 표시가 일치한다.

### 5-3. 레이드 숙제 완료 연동

1. `index.html`에서 레이드 숙제를 완료 체크한다.
2. `overview.html`에서 해당 캐릭터 레이드 완료 상태를 확인한다.
3. `raid.html` 주간 일정 또는 공유 뷰어 레이드 현황에서도 상태가 반영되는지 확인한다.
4. 완료 해제 후 다시 모든 페이지를 확인한다.

성공 기준:

- 완료 상태가 페이지 간 일관된다.
- 완료 저장 후 바로 페이지 이동해도 상태가 되돌아가지 않는다.

### 5-4. 파티 연동 배지와 팝업

1. `raid.html`에서 해당 캐릭터가 포함된 파티를 만든다.
2. `index.html`로 돌아온다.
3. 레이드 숙제에 파티연동 배지가 표시되는지 확인한다.
4. 배지를 클릭한다.
5. 해당 캐릭터가 포함된 파티만 팝업에 보이는지 확인한다.
6. 파티 이동 버튼을 누른다.

성공 기준:

- 다른 캐릭터만 포함된 파티는 팝업에 보이지 않는다.
- 파티 이동 버튼은 `raid.html`의 해당 파티/탭으로 이동한다.
- 임시 파티 연동이면 `임시파티연동` 배지가 표시된다.

## 6. index.html 재화 시나리오

### 6-1. 재화 추가와 직접 입력

1. 재화 현황에서 `추가`를 누른다.
2. 재화 이름과 아이콘을 선택한다.
3. 저장한다.
4. 재화 카드가 `- / 재화명 + 수량 입력 / +` 형태로 보이는지 확인한다.
5. 수량 입력칸에 숫자를 직접 입력한다.
6. Enter 또는 포커스 이동으로 저장한다.
7. 새로고침 후 수량이 유지되는지 확인한다.

성공 기준:

- alert/prompt 입력창이 뜨지 않는다.
- 숫자 입력칸에 브라우저 기본 상하 화살표가 보이지 않는다.
- 재화명 텍스트가 잘리지 않는다.
- `최종 수정 날짜: YYYY.MM.DD`가 재화 현황 헤더 중앙/우측 영역에 표시된다.

### 6-2. +/- 버튼과 최종 수정 날짜

1. `+` 버튼을 여러 번 누른다.
2. `-` 버튼을 여러 번 누른다.
3. 수량이 0 아래로 내려가지 않는지 확인한다.
4. 최종 수정 날짜가 갱신되는지 확인한다.

성공 기준:

- +/- 버튼과 중앙 카드가 따로 노는 것처럼 보이지 않는다.
- 값 변경 직후 UI가 즉시 갱신된다.
- 날짜가 작은 텍스트로 칩 옆에 붙지 않는다.

### 6-3. 수정 모드

1. 재화 현황 헤더의 `수정` 버튼을 누른다.
2. 각 재화 카드 우측에 수정 아이콘이 표시되는지 확인한다.
3. +/- 버튼은 사라졌는지 확인한다.
4. 수정 아이콘을 누른다.
5. 이름, 수량, 아이콘을 변경하고 저장한다.
6. 다른 화면 영역을 클릭했을 때 수정 모드가 자동으로 꺼지는지 확인한다.

성공 기준:

- 수정 모드와 일반 모드가 명확히 구분된다.
- 수정 아이콘을 누르지 않고 재화 섹션 바깥을 클릭하면 토글이 꺼진다.
- 저장 후 목록에 즉시 반영된다.

### 6-4. 삭제 모드

1. 재화 현황 헤더의 `삭제` 버튼을 누른다.
2. 각 재화 카드 우측에 삭제 아이콘이 표시되는지 확인한다.
3. +/- 버튼은 사라졌는지 확인한다.

- 원본 초기화 후 임시 표시가 사라진다.

## 15. 페이지 간 연동 종합 시나리오

### 15-1. 완료 상태 왕복

1. `index.html`에서 레이드 숙제를 완료한다.
2. `overview.html`에서 완료 상태를 확인한다.
3. `raid.html` 공유 뷰어에서 상태 표시를 확인한다.
4. `raid.html`에서 완료 해제한다.
5. `index.html`과 `overview.html`에서 해제 상태를 확인한다.

성공 기준:

- 완료 상태가 한 방향으로만 반영되지 않고 왕복 동기화된다.

### 15-2. 파티 변경 왕복

1. `raid.html`에서 파티 멤버를 변경한다.
2. `index.html` 파티연동 팝업을 확인한다.
3. `overview.html` 파티연동 팝업을 확인한다.
4. `parties.html` 파티 카드를 확인한다.

성공 기준:

- 네 페이지에서 같은 파티 정보를 본다.

### 15-3. 부계정 그룹 왕복

1. 본계정과 부계정에 각각 캐릭터를 둔다.
2. `index.html` 계정 탭을 확인한다.
3. `overview.html` 필터와 본문을 확인한다.
4. `raid.html` 공유 뷰어 레이드 현황 필터와 본문을 확인한다.
5. `parties.html` 필터를 확인한다.

성공 기준:

- 부계정 캐릭터가 본계정 캐릭터와 중복 표시되지 않는다.
- 부계정 그룹명이 페이지마다 이해 가능한 형태로 표시된다.

### 16-1. 데스크톱 좁은 화면

성공 기준:

- 버튼과 텍스트가 겹치지 않는다.
- 모달 내부 텍스트가 잘리지 않는다.

### 16-2. 모바일 폭

1. 개발자도구에서 모바일 폭으로 전환한다.
2. 5개 주요 페이지를 모두 확인한다.

4. `raid.html` 일정 모달 파티 목록과 미리보기가 세로로 내려가는지 확인한다.

성공 기준:

- 가로 스크롤 없이 주요 기능을 사용할 수 있다.
- 텍스트가 버튼/카드 밖으로 넘치지 않는다.

## 17. 데이터 유지와 초기화 시나리오

### 17-1. 새로고침 유지

다음 항목을 각각 변경한 뒤 새로고침한다.

- 계정 순서
- 캐릭터 순서
- 마지막 선택 캐릭터
- 원정대 숙제 완료
- 캐릭터 일일 숙제 완료
- 캐릭터 주간 숙제 완료
- 하위 숙제 순서
- 레이드 숙제 골드 수령 토글
- 재화 수량/아이콘/이름
- 코어 등급/필수/질서 코어명
- 파티 멤버 슬롯
- 주간 일정
- 공지사항
- 임시 파티 변경

성공 기준:

- 저장되어야 하는 데이터는 유지된다.
- 주차에 따라 초기화되어야 하는 데이터만 초기화된다.

### 17-2. 일일/주간 초기화

1. 일일 숙제 완료 시간을 지난 일일 초기화 기준 이전 값처럼 만든다.
2. 페이지를 다시 로드한다.
3. 일일 숙제가 초기화되는지 확인한다.
4. 주간 숙제는 주간 초기화 기준에 맞게만 초기화되는지 확인한다.

성공 기준:

- 일일/주간 초기화 기준이 섞이지 않는다.
- 카운트형 숙제의 `count_current`도 함께 초기화된다.
