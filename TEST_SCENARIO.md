# Test Scenario

## 0. 테스트 전 준비

1. GitHub Pages 캐시 제거
2. 브라우저 개발자도구 Console 열기
3. Supabase `schema.sql` 최신 적용 여부 확인
4. 테스트 중 빨간 콘솔 에러가 없어야 함

## 1. 공통 로딩 테스트

대상:

- `index.html`
- `core.html`
- `raid.html`
- `overview.html`
- `parties.html`

성공 기준:

- 상단 메뉴 정상
- CSS 깨짐 없음
- 화면 하단 코드 노출 없음
- `</html>` 뒤 코드 없음
- 콘솔 에러 없음

## 2. index.html

### 계정/부계정

- 본계정 추가
- 부계정 추가

- 부계정 표시용 `...` 텍스트가 나오지 않는지 확인

### 캐릭터

- 캐릭터 추가/수정/삭제
- 아이콘 선택/제거
- 새로고침 후 유지

### 원정대 숙제

- 추가/수정/삭제
- 아이콘 선택/제거
- 체크 상태 저장
- 복제 기능 확인

### 일일/주간 숙제

- 추가/수정/삭제
- 수정 버튼 클릭 시 `setTaskIcon is not defined` 에러 없어야 함
- 아이콘 선택 위치 정상
- 체크 상태 저장

### 레이드 숙제

- 유통골드/귀속골드/더보기 골드 분리 표시
- 각 골드 hover 효과 확인

- 레이드 숙제 수정 버튼이 보이지 않아야 함
- 파티연동 hover 효과 확인
- 파티연동 클릭 시 해당 캐릭터가 속한 파티만 표시
- 파티 이동 버튼 클릭 시 `raid.html` 해당 파티로 이동

### 재화

- 금액 수정
- 아이콘 선택
- 중앙에 `최종 수정 날짜: YYYY.MM.DD` 표시
- 날짜가 작게 옆에 붙어 있지 않아야 함
